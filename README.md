📝 AiTODO（爱TODO）产品需求文档 (PRD)
项目版本： v1.0.0 (MVP)

开发框架： Flutter (Android / iOS / Windows)

定位： 一款融合 AI 深度解析、跨端隐私同步与极简美学的全能待办助手。

1. 产品愿景
通过 AI 消除“记录任务”的摩擦力。让用户用最自然的语言记录生活（爱生活），用最智能的方式管理效率（爱 TODO）。

2. 核心功能模块
2.1 AI 智能引擎 (AI Dispatcher)
这是项目的灵魂，支持三种模式切换：

内置模式： 默认集成 Gemini Flash API，提供开箱即用的 NLP 体验。

极客模式 (BYOK)： 用户可填写自己的 API Key（支持 OpenAI/DeepSeek 协议），自定义模型参数。

隐私模式 (Local)： 集成本地轻量化模型（如 Gemma-2B），确保在完全断网及隐私敏感场景下仍能解析任务。

功能点： * 语义提炼： 从“明早八点提醒我带咖啡”提取出 时间: 08:00、内容: 带咖啡。

任务拆解： 输入“策划团建”，AI 自动生成 5 个子任务。

2.2 任务与日历管理
视图切换： 支持列表视图、看板视图及日历视图（周/月）。

多端适配：

移动端： 侧重单手操作、快速录入（语音+文字）。

Windows 端： 侧重多任务查看、侧边栏常驻、全局快捷键呼出。

提醒系统： 跨平台本地通知，支持“强提醒”模式，确保关键任务不遗漏。

2.3 全球化云同步
双云架构： * Global： 默认对接 Supabase / Firebase，遵循 GDPR 协议。

China： 可切换至 LeanCloud / 境内 Supabase 节点，确保访问速度与 PIPL 合规。

离线优先： 使用 Isar 数据库，所有操作先落本地，网络恢复后后台静默同步。

2.4 数据统计与分析
热力图： 展示用户每月的打卡活跃度。

生产力报告： 统计任务完成率、各分类时间占比、专注时段分析。

3. 技术路线图 (Tech Stack)
维度	技术选型
前端	Flutter 3.x (Dart)
本地存储	Isar Database (高性能、强类型)
AI 框架	Google Generative AI (Cloud) + MediaPipe (Local)
状态管理	Riverpod 或 Bloc
统计图表	fl_chart
推送通知	flutter_local_notifications + FCM
4. 非功能性需求
4.1 性能要求
AI 响应时间（云端）应控制在 2 秒内。

应用冷启动速度在 Windows 端应小于 1.5 秒。

4.2 UI/UX 风格
风格： 现代简约，带有呼吸感的微动效。

品牌色： 极客蓝（#2196F3）渐变至 活力桃（#FF4081）。

图标： 包含“勾选”与“心形”元素的融合 Logo。

5. 合规性需求 (Compliance)
Google Play： 必须包含 AI 生成内容声明，提供账号注销入口。

中国市场： 国内版必须包含《隐私政策》和《用户协议》弹窗，且数据存储于境内服务器。

6. 未来规划 (Roadmap)
v1.1: 增加番茄钟功能。

v1.2: 支持共享清单（多人协作）。

v2.0: 深度集成系统层，如 Windows 桌面小组件和 iOS 灵动岛。

结语： AiTODO 的核心竞争力在于**“选择权”**——用户可以选择自己信任的 AI 引擎，选择自己信任的云端。# aitodo_by_flutter
